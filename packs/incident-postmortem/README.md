# Incident Postmortem Pack

SRE perspective focused on blameless postmortems — reviews for root cause depth and whether action items will actually prevent recurrence.

## Quick Start

```bash
# Review a postmortem for quality and blamelessness
cat postmortem.md | greybeard analyze --pack incident-postmortem

# Learn how to write effective postmortems
cat postmortem.md | greybeard analyze --pack incident-postmortem --mode mentor
```

## Focus Areas

- **Blamelessness**: Are we blaming individuals or systems?
- **Root Cause Depth**: Did we find the systemic condition, not just the trigger?
- **Action Items**: Are they specific, owned, and actually preventative?
- **Timeline**: Is the sequence of events complete and reconstructable?

## When to Use This Pack

Use this pack when writing or reviewing postmortems after incidents — to ensure you're learning systemic lessons and making changes that will actually prevent recurrence.
